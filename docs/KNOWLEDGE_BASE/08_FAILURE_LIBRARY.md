## 1. Proposito

Crear una biblioteca de fallos frecuentes y averias habituales basada en experiencia real y conocimiento validado.

## 2. Contenido previsto

- Sintoma.
- Causa probable.
- Componentes afectados.
- Marca/modelo si aplica.
- Diagnostico recomendado.
- Fotos o ejemplos.
- Repuestos compatibles.
- Procedimiento sugerido.
- Observaciones internas.

## 3. Relacion con ITI

Las inspecciones pueden alimentar la biblioteca cuando una averia se repite y el conocimiento es validado por SAT u oficina tecnica.
